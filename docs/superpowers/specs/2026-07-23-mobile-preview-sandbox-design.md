# 㤫者商城手机预览沙盒设计

## 目标

为“㤫者商城”用户端建立仅限开发环境的手机预览沙盒，使产品负责人可以通过同一 Wi‑Fi 在手机浏览器中查看并操作完整 H5 流程，同时不启动、不修改、不写入真实后端。

成功标准：

1. 手机可通过局域网地址和 `preview=1` 参数进入预览。
2. 正常地址、微信正式环境和生产构建不启用测试数据。
3. 预览数据覆盖首页、甄客验详情、试用、购物车、订单、物流和个人中心。
4. 预览操作只改变当前浏览器内的临时状态，刷新后可恢复。
5. 页面明确提示“本地预览数据”，避免与真实业务数据混淆。

## 安全边界

预览沙盒必须同时满足以下条件才可启用：

- 当前为开发构建；
- 主机名为 `localhost`、`127.0.0.1` 或局域网 IPv4 地址；
- 地址参数包含 `preview=1`。

预览模式不得：

- 写入数据库；
- 调用真实支付、退款、物流或发布接口；
- 修改后端接口、字段、状态、权限；
- 在生产构建中生效；
- 伪装成真实用户数据。

## 手机访问

开发服务器继续监听局域网地址。手机与 Mac 连接同一 Wi‑Fi 后，通过以下形式访问：

`http://<Mac局域网地址>:8000/?preview=1`

如果局域网地址变化，只更新访问地址，不修改业务代码。

## 预览数据

测试数据使用独立前端模块维护，不混入正式接口服务。数据必须严格复用现有类型、字段和状态值。

### 首页

- 默认混合展示购买甄客验、线上试用和线下试用。
- 四个固定分类均有内容：户外、运动服装、健康产品、生活优选。
- 初始不选择分类，点击分类后筛选，再次点击恢复全部内容。
- 卡片覆盖有图、长标题、不同互动数量和不同试用名额。

### 甄客验详情

- 购买来源：展示商品质量、物流服务、服务态度三个真实字段。
- 试用来源：不展示上述三个评分。
- 作者身份统一为“甄客”。
- 包含评论、回复、绿色“有用”状态和关联商品。
- 不生成虚构标题字段，不显示关注按钮或购买来源标签。

### 试用详情

- 线上试用：申请、审核、发货、确认收货、发布甄客验。
- 线下试用：申请、审核、发布甄客验，不出现物流步骤。
- 展示目标人数、已领取、剩余人数、截止日期和只读试用方式。
- 不出现三天倒计时、处罚或配送方式选择。

### 购物车与订单

- 购物车覆盖单商品、多商品、数量变化和空购物车。
- 普通购买订单覆盖待付款、待发货、待收货、待发布和退款状态。
- 线上、线下试用只与普通订单统一卡片视觉，底层预览数据仍保持不同来源。
- 物流详情覆盖备货、运输中、已签收和异常状态。
- 支付、退款等危险操作只模拟界面状态，不调用真实接口。

### 个人中心

- “我的甄客验”位于订单区之前。
- 编辑资料与收货地址并排。
- 订单状态入口以“全部订单”为第一项。
- 不设置独立“我的试用”一级入口。
- 购买订单与试用申请可以统一卡片展示，但保留来源标识和各自状态映射。

## 预览交互

预览状态保存在当前页面内存中。必要时可使用仅带预览命名空间的 `sessionStorage` 保持同一次巡检过程；不得写入正式业务使用的本地存储键。

支持的模拟操作：

- 切换分类；
- 打开甄客验、试用、商品、购物车和个人中心；
- 标记“有用”；
- 申请线上或线下试用；
- 加入购物车和调整数量；
- 模拟订单状态变化；
- 查看不同物流状态；
- 查看购买和试用甄客验的评分差异。

所有会产生真实外部副作用的按钮在预览模式下必须显示“预览模拟”提示。

## 巡检入口

预览模式增加轻量的“巡检菜单”，仅在 `preview=1` 时显示。菜单按业务流程提供入口：

1. 首页全部内容；
2. 购买甄客验详情；
3. 试用甄客验详情；
4. 线上试用招募；
5. 线下试用招募；
6. 购物车；
7. 各状态订单；
8. 物流详情；
9. 我的甄客验；
10. 个人中心。

巡检菜单用于快速定位状态，不替代正式页面导航。

## 架构

建议拆分为三个边界清晰的单元：

1. `previewMode`：只负责判断是否允许启用预览。
2. `previewFixtures`：只负责提供符合现有类型的测试数据。
3. `previewState`：只负责在浏览器内模拟操作和状态流转。

正式页面继续使用现有服务。页面读取数据时先判断预览模式：

- 预览模式：从预览状态读取；
- 正常模式：继续调用现有真实接口。

不得把预览判断散落到每个按钮中，应通过统一的数据和动作适配层隔离。

## 错误处理

- 手机无法访问时，优先检查同一 Wi‑Fi、Mac 防火墙和局域网地址。
- 预览数据缺失时，显示明确的预览错误卡片，不回退调用真实接口。
- 预览动作遇到不支持状态时，保持当前数据并显示说明。
- 正常模式的错误处理保持现状。

## 测试

### 自动测试

- 只有开发环境、局域网主机和 `preview=1` 同时满足时启用。
- 生产环境永远禁用。
- 首页默认混合数据及四分类筛选正确。
- 购买甄客验显示三评分，试用甄客验不显示。
- 线上与线下试用流程步骤不同。
- 订单和试用数据源不混用。
- 预览支付、退款、物流等操作不调用真实服务。
- 正常模式继续调用原服务。

### 手机人工测试

- iPhone 或 Android 浏览器可打开局域网预览地址。
- 375px、390px、430px 常见宽度无横向溢出。
- 底部导航、悬浮购物车和弹层不被安全区域遮挡。
- 从巡检菜单可进入所有关键界面。
- 完整走查首页、详情、试用、购物车、订单和个人中心。

## 分阶段实施

每个阶段最多修改三个文件：

1. 手机局域网预览条件与入口。
2. 预览数据模块和首页、详情、试用接入。
3. 购物车、订单、物流和个人中心预览状态。
4. 巡检菜单、手机尺寸验证和完整回归。

每个阶段均先写失败测试，再实现，最后运行结构测试和构建。
